# ナレッジ階層化システム設計書

## 📋 システム概要

### 目的
INBOXのファクト（アポメモ）を自動的にタグ付けし、実用的な検索・活用ができるナレッジシステムを構築する

### 検索・活用のシナリオ
**「⭕️業界、⭕️人規模の組織でこういう話聞いたけど、近しい事例とか考え方、打ち手の実績、知っておくべき他社の事例や知識あるかな？」**

## 🎯 ナレッジの観点

### 1. 業界の特徴
- **業界特有の働き方**: 長時間労働、現場重視、プロジェクト型など
- **市場感**: 競争環境、顧客特性、業界トレンド
- **人材構成**: 年齢層、性別比率、専門性の分布
- **コミュニケーション文化**: 上下関係、意思決定プロセス、情報共有スタイル
- **よくある問題**: 業界特有の課題、失敗パターン

### 2. 組織特徴
- **規模別の特徴**: 大企業・中堅・ベンチャー別の特徴
- **働き方**: フレックス、リモート、現場勤務の比率
- **人材構成**: 年齢・性別・職種の分布
- **コミュニケーション**: 会議文化、報告ルート、意思決定
- **よくある問題**: 規模特有の課題、失敗パターン

### 3. 打ち手・施策
- **具体的な施策**: 実際に実施された取り組み
- **実施方法**: ステップ、期間、予算、体制
- **成功・失敗**: 結果とその要因
- **他社事例**: 類似業界・規模での成功事例
- **知っておくべき知識**: 法的要件、業界標準、ベストプラクティス

## 🔄 処理フロー（改訂版）

### 1. INBOX監視
- 新規ファイルの自動検出
- ファイル名から基本情報抽出（企業名、日付、テーマ）

### 2. 自動タグ付け
- 業界・規模・階層の自動判定
- 問題・課題の自動抽出
- 施策・打ち手の自動識別
- 結果・効果の自動分類

### 3. ナレッジ生成
- 業界・組織特徴の抽出
- 問題・課題の整理
- 打ち手・施策の詳細化
- 成功・失敗要因の分析

### 4. 関連性発見
- 類似業界・規模での事例検索
- 他社の成功事例との関連付け
- ベストプラクティスの抽出

## 📊 検索・活用パターン

### パターン1: 業界・規模別の課題解決
**「建設業界の大企業で女性活躍を推進したいが、どんな課題があるか？」**
→ 建設業界の特徴、大企業の組織特徴、女性活躍の課題、成功事例を検索

### パターン2: 他社事例の参考
**「IT業界のベンチャーで働き方改革を進めているが、他社はどうしているか？」**
→ IT業界の特徴、ベンチャーの組織特徴、働き方改革の事例、成功要因を検索

### パターン3: 施策の効果検証
**「製造業でメンター制度を導入しようとしているが、効果はどうか？」**
→ 製造業の特徴、メンター制度の事例、成功・失敗要因、効果測定結果を検索

### パターン4: 業界横断的な知見
**「どの業界でも通用する組織風土改革の手法は？」**
→ 複数業界での組織風土改革事例、共通の成功要因、業界別の違いを検索
